## The Drips Wave Point System

Accensa is an active participant in the Drips Wave funding mechanism. We manage an ongoing pipeline of issues across all components (contracts, indexer, dashboard, and SDK).

Contributors are rewarded based on issue complexity. We strictly map our issue labels to the Drips points system.

### Complexity Labels

1. **Trivial (100 points):** Small, clearly bounded changes with obvious acceptance criteria. Example: adding typed error classes to the SDK.
2. **Medium (150 points):** Standard features that touch multiple files or systems. Example: building a new table in the Next.js dashboard with filtering.
3. **High (200 points):** Integrations and core architectural work. Example: building the database schema backfill mechanism for the Go Indexer.

## Guidelines for Pull Requests

- **Strict Acceptance Criteria:** Do not open a PR unless you can explicitly check off every requirement in the issue's acceptance criteria.
- **Run Local Checks:** We enforce strict CI checks. Before opening a PR, always run your tests locally (e.g., `cargo test`, `pnpm build`, `go build`). If a PR fails CI because of a basic formatting or linting error, it will be delayed.
- **One PR per Issue:** Do not combine multiple unrelated fixes into a single PR. Keep diffs small and readable.
